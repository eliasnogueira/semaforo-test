Semáforo

Esta aplicação foi criada para a apresentação no Campus Party 2014 com o título "Nunca foi tão fácil testar". A palestra apresenta como é o desenvolvimento de uma aplicação de forma a sempre pensar (e aplicar) testes. Nesta linha foi seguido alguns conceitos de desenvolvimento ágil a aplicação de TDD - Test Driven Development, BDD - Behavior Driven Development e Automação da Aceitação (interface gráfica).

Todo o código desenvolvido foi criado com fins educacionais. Os mesmos podem não estar 100% de acordo com os padrões de desenvolvimento adotados em um desenvolvimento profissional.

Estrutura do projeto

O projeto foi criado no Eclipse IDE como um projeto Maven. Ele possui a seguinte estrutura

_pasta src: _contem uma classe java simulando a transição dos estados de um sinal semafórico (semáforo), Este não possui nenhum time ou scheduler. O intuito foi puramente ilustrativo para fins didáticos.

_pasta test: _contém todos os testes separados em três níveis:

  • unit: testes unitários onde foi apresentado a técnica de TDD
  • integration: testes de integração utilizando a técnica de BDD
  • acceptance: testes de aceitação

Frameworks utilizados

Para o desenvolvimento de testes unitários foi utilizado o JUnit 4. Para o desenvolvimento dos testes de integração com BDD foi utilizado a ferramenta JBehave * Para o desenvolvimento dos testes de aceitação foi utilizado o Selenium Webdriver *

  • Para instalar o plugin do JBehave no Eclipse IDE acesse a seguinte página http://jbehave.org/download.html
  • Para utilizar o Webdriver no Eclipse adicione as bibliotecas java e do selenium-server manualmente através da pagina de downloads do Selenium ou via Maven.
